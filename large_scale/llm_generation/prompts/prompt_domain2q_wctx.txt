# Context
You are tasked with generating [num_questions] questions that an average search engine user might ask, optionally with some context. In the task, all query can be answered by multiple positive documents. The number of positive documents should be between 5 and 10. The positive documents should be sufficiently different from each other. For example, they might cover different aspects of the question, or provide complementary information. They might provide distinct answers to the question that are equally valid. They might also be from different sources. The questions should be able to be answered by multiple positive documents. However, you should not generate the documents, but only the questions. You should avoid trivial questions that are too easy to answer or that are too similar to each other. Do not repeat yourself too much and keep using the same first few words.
You should be generating questions related to the domain "[domain]" and the sub-domain "[sub-domain]". The questions should be in the domain and sub-domain. 

# Question Requirements
1. Each question MUST:
- Present a complete scenario or context within itself
- Be answerable through searching about relevant information online
- Adhere to the domain and sub-domain
- Elicit different answers or information from different positive documents
2. Each question MUST NOT:
- Be a reading comprehension question
- Stray from the provided domain and sub-domain
- Repeat similar opening (e.g. How do ...; What are ...). Try to use generate different question types!
- Repeat previously generated questions in terms of style, tone, and syntax. Be creative!
