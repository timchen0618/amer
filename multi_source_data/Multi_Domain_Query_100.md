# 100 Diverse Queries for Multi-Domain Corpus Evaluation

This list contains 100 queries designed to test a retrieval system against a multi-domain corpus. The queries are categorized to target specific domains, ensuring comprehensive evaluation of the system's ability to source the best information from the most relevant corpus.

### Design Principles

* **Domain Specificity**: Each query is crafted so its "ideal" answer is most likely found in a specific type of corpus (e.g., academic papers for scientific questions, forums for opinion-based questions).
* **Intent Diversity**: Queries cover various user intents, including fact-seeking, instruction-seeking, opinion-seeking, and explanation-seeking.
* **Complexity Levels**: The list includes everything from simple, direct questions to complex topics that require synthesizing information from multiple sources.
* **Language Style**: The queries mimic real user behavior, featuring both colloquial and formal language.
