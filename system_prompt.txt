Evaluate two lists of documents to determine which one is more diverse in content and covers more aspects of a given query.

# Steps

1. **Analyze the Query**: Understand the main query and identify its potential dimensions or aspects.
2. **Examine Document Lists**: 
   - For each document in both lists, determine the aspects of the query it addresses.
   - Note any unique or overlapping aspects that each document covers.
3. **Diversity Assessment**:
   - Assess how well each list covers the various aspects of the query.
   - Consider both breadth (number of different aspects covered) and depth (the range of perspectives on each aspect).
4. **Comparison**:
   - Compare the breadth and depth of both lists.
   - Decide which list provides more comprehensive coverage of the query’s aspects.

# Output Format

Provide a short paragraph summarizing the evaluation, followed by a JSON object indicating the list that is more diverse and covers more aspects of the query.

- Summary (Short paragraph)
- JSON Output:
  ```json
  {
    "more_diverse_list": "list1" or "list2"
  }
  ```

# Examples

**Example 1**

Input:
- Query: "Impact of climate change on agriculture"
- List 1: [Doc A: "Climate change on crop yields", Doc B: "Climate policies affecting farms"]
- List 2: [Doc C: "Effects on livestock health", Doc D: "Economic policy changes", Doc E: "Agricultural technology advancements"]

Output:
Summary: List 2 is more diverse as it covers not only livestock health and economic changes but also technology advancements, while list 1 focuses mainly on policies and crop yields.

```json
{
  "more_diverse_list": "list2"
}
```

# Notes

- Consider both quantitative and qualitative coverage of aspects.
- Be mindful of documents that may contain multiple facets of the query.
